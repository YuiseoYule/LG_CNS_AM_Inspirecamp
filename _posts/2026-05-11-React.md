# Today's lecture


## React: Component-Based Library

- **Library vs. Framework**: React is a **Library** because the developer maintains control and "calls" React features, rather than the software dictating the entire structure (Inversion of Control).
- **Core Workflow**: `Props` (Input) → `Component` (Function/Template) → `Element` (Output/UI).

---

### 1. Components & Elements
- **Component**: A reusable "building block" or "mold" (usually a `.jsx` file). Technically, it is a JavaScript function.
- **Element**: The smallest unit in React; it describes what you see on the screen after a component is executed.
- **Rendering**: The process where React elements are transformed into actual DOM nodes that the browser displays.

### 2. Props (Properties)
- **Definition**: Inputs passed into a component to customize its output.
- **Analogy**: Attributes in HTML (`attr="value"`) are conceptually the same as `Props` in React.
- **Benefit**: High **reusability**. One component can generate infinite unique elements based on different props.

### 3. Development Setup
- **CRA**: *Create React App*—a CLI tool used to quickly set up a functional React development environment.

### Summary Table
| Concept | Technical Meaning | Analogy |
| :--- | :--- | :--- |
| **Component** | JavaScript Function | Lego Mold / Blueprint |
| **Props** | Function Arguments / Attributes | Specific Data (Color, Size) |

## React Environment and Architecture

- **SPA (Single Page Application)**: A web application that loads only a single HTML page. Content is dynamically updated by swapping **Components** rather than loading new pages.
- **Key Tools**:
    - `npm`: Package manager for installing/managing libraries.
    - `npx`: Package runner used to execute `create-react-app` without permanent installation.

---

### 1. Project Directory Layout
- **public/index.html**: The only physical HTML file. Contains `<div id="root"></div>`, which serves as the mounting point for the entire app.
- **src/index.js**: The "bridge" between React and the Browser. It renders the main component into the `root` ID.
- **src/App.js**: The root **Component**. A JavaScript function that returns UI structure (JSX).

### 2. Component Workflow
1. **Definition**: Create a function-based component in a `.js` or `.jsx` file.
2. **Export**: Use `export default ComponentName` to make it available.
3. **Import**: Bring the component into `index.js` using `import`.
4. **Rendering**: React takes the component's output (**Element**) and injects it into the DOM.

### 3. Key Attributes (Recap)
- **Class**: Used primarily for **CSS** styling.
- **ID**: Used primarily for **Scripting** (e.g., React targeting the `root` element).

### 4. Running the Project
- **Command**: `npm start`
- **Environment**: Starts a local development server (usually at `localhost:3000`).
- **Note**: If the terminal fails to recognize commands, a restart of the IDE (VS Code) is recommended to refresh environment variables.

---

## React Components and JSX

- **JSX (JavaScript XML)**: An extension of JavaScript that allows you to write HTML-like structures directly inside script files. It is transpiled into standard JavaScript for the browser.
- **File Convention**: Component filenames and function names should start with an **Uppercase** letter (e.g., `Login.jsx`).



---

### 1. Component Structure (Functional)
- **Definition**: Defined using ES6 **Arrow Functions** for modern syntax.
- **Return**: Every component must `return` a single top-level JSX element.
- **Export/Import**: Components are exported (`export default`) and then imported into other files (like `index.js`) to be rendered.

### 2. Styling and Reserved Keywords
- **className**: Because `class` is a reserved keyword in JavaScript, React uses `className` to apply CSS classes.
- **CSS Import**: Stylesheets are imported directly into the script: `import './Login.css'`.

### 3. State Management (Hooks)
- **useState**: A React Hook used to manage "State" (data that changes over time).
- **Two-Way Binding**: Synchronizes the UI input fields with JavaScript variables so the app can track what the user types in real-time.

### 4. Event Handling
- **Handlers**: Functions defined within the component to process user actions (e.g., `handleSubmit`).
- **event.preventDefault()**: A critical method used in form submission to stop the browser's default behavior (reloading the page/event bubbling).

### 5. Backend Integration
- **Data Format**: React typically communicates with backends (e.g., Spring Boot) using **JSON**.
- **Communication Libraries**:
    - **Axios**: A popular promise-based HTTP client.
    - **Fetch API**: A built-in browser interface for fetching resources.
