# Today's lecture





### 1. Style & UI Logic in React
React transitions the UI based on internal "State" rather than manual DOM manipulation.
*   **JSX Attributes:** Uses `className` instead of `class` to avoid conflicts with JavaScript reserved words.
*   **Inline Styling:** Styles can be defined as JavaScript objects `{{ key: value }}` for dynamic UI changes.
*   **State-Driven UI:** UI elements (like error messages) are conditionally rendered based on the current state (e.g., `isValid` or `errorMessage`).

---

### 2. Asynchronous Communication (Axios)
To connect the Front-end with the Back-end, developers use HTTP clients.
*   **Axios:** A library for making HTTP requests (GET, POST, etc.).
*   **Async/Await:** Ensures the application waits for the server response (e.g., login validation) without blocking the main execution thread.
*   **Token Management:** Successful authentication often returns a **JWT (JSON Web Token)**, which is stored in the browser's `localStorage` to maintain the user session.

---

### 3. Client-Side Routing (SPA Architecture)
In a Single Page Application (SPA), "navigation" is the process of swapping components without a full page reload.
*   **React Router DOM:** The primary library for managing browser URLs in React.
*   **Path Mapping:** Links a specific URL (e.g., `/welcome`) to a specific Component (e.g., `<Welcome />`).
*   **Hooks for Navigation:**
    *   `useNavigate`: A function used to redirect users programmatically (e.g., after a successful login).
    *   `useLocation`: A tool to retrieve data (state) passed from the previous page.

---

### 4. Component Architecture & Hooks
*   **Components:** Modular, reusable UI building blocks (e.g., Login Box, Navbar).
*   **Hooks:** Functions (e.g., `useState`, `useEffect`) that allow functional components to manage complex logic and lifecycles.
*   **Props & State:** 
    *   **State:** Data managed *inside* a component.
    *   **Props:** Data passed *from parent to child* components.
